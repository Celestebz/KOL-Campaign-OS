# KOL Campaign OS External Agent

## Default Workflow

Use KOL Campaign OS end to end through HTTP APIs:

```text
Campaign/product brief
-> KOL Strategy
-> mark Strategy ready
-> Video Evidence Finder
-> Raw Candidates generated by system
-> human approval in UI
```

Do not use the old Cycle x Route / Subagent Hybrid / direct Raw Candidate import workflow unless the user explicitly asks for legacy behavior.

## Finder Default

Use **Video Evidence Finder v1** by default.

The final goal is still KOL profile links:

```text
author_profile_url = final KOL profile link
video_url = evidence used to justify the KOL candidate
```

Profile URLs are identity/leads. They are not enough evidence by themselves.

The agent may discover profiles first, but must then find target-platform videos from those profiles before importing anything.

## Required Finder Boundary

```text
execution_mode = video_evidence_finder
discovery_scope = target_platform_only
discovery_route = target_platform_first
allow_cross_platform_evidence = false
target_platform = evidence_platform
```

Do not use cross-platform routes in v1.

Examples:

- Instagram task -> import Instagram Reels/Posts only.
- YouTube task -> import YouTube videos/Shorts only.
- TikTok task -> import TikTok videos only.

Do not use YouTube evidence for an Instagram Finder task.

## Required Finder API Flow

Create or continue a Finder task:

```http
POST {base_url}/api/finder-tasks
Content-Type: application/json

{
  "strategy_id": 1,
  "execution_mode": "video_evidence_finder",
  "target_platforms": ["instagram"],
  "discovery_scope": "target_platform_only",
  "allow_cross_platform_evidence": false,
  "limit_per_platform": 10
}
```

Import video evidence:

```http
POST {base_url}/api/finder-tasks/{finder_task_id}/video-evidence/import
Content-Type: application/json

{
  "videos": [
    {
      "video_url": "https://www.instagram.com/reel/xxxx/",
      "target_platform": "instagram",
      "evidence_platform": "instagram",
      "source_signal": "use_case",
      "source_query": "vocal processor live performance",
      "title": "optional visible title",
      "author_name": "creator handle or display name",
      "author_profile_url": "https://www.instagram.com/creator/",
      "evidence_reason": "Why this video proves the creator may fit the KOL search."
    }
  ]
}
```

Crawl imported videos:

```http
POST {base_url}/api/videos/crawl
Content-Type: application/json

{ "videoIds": [123] }
```

Run evidence scoring:

```http
POST {base_url}/api/finder-tasks/{finder_task_id}/evidence-analysis
```

Generate Raw Candidates:

```http
POST {base_url}/api/finder-tasks/{finder_task_id}/generate-candidates-from-evidence
```

Do not call approval APIs.

## Video Evidence Rules

Accepted `video_url` formats:

- `https://www.youtube.com/watch?v=...`
- `https://www.youtube.com/shorts/...`
- `https://youtu.be/...`
- `https://www.instagram.com/reel/.../`
- `https://www.instagram.com/p/.../`
- `https://www.tiktok.com/@handle/video/...`

Do not import:

- `https://www.instagram.com/creator/` as `video_url`
- `https://www.youtube.com/@channel` as `video_url`
- `https://www.tiktok.com/@handle` as `video_url`

Those profile links belong in `author_profile_url`.

If only a profile is found and no relevant video is found, keep it in:

```text
profile_leads_needing_video_evidence
```

Do not import it.

## Strategy And Campaign Flow

If no ready Strategy exists:

1. Inspect campaigns with `GET {base_url}/api/campaigns`.
2. Ask the user which Campaign to use, or create one after collecting enough brief details.
3. Create or improve a Strategy.
4. Mark the Strategy ready if the user wants Finder to run.
5. Read the Agent Brief.
6. Run Video Evidence Finder.

If a ready Strategy exists:

1. Read the Agent Brief.
2. Confirm current target platform if it was not explicitly selected in the current request or UI payload.
3. Continue existing `video_evidence_finder` task if one exists.
4. Otherwise create a new `video_evidence_finder` task.
5. Search and import target-platform video evidence.

Do not ask for old Quick / Standard / Full Cycle intensity for Video Evidence Finder v1. The new default first-pass signals are:

```text
category
use_case
native_platform
```

These are signal sources, not mandatory sequential cycles.

## Required Inputs

Use defaults when not specified:

- `base_url`: `http://localhost:5001`
- Database: MySQL through app APIs only

Collect or infer:

- product / brand / campaign
- target market and language
- current target platform
- target count
- follower/tier preference if important
- competitors or exclusions if supplied

Do not silently use `Default Campaign` unless the user confirms it.

Do not print or expose API tokens.

## Hard Boundaries

- Do not approve candidates into KOL Master or Campaign KOL.
- Do not write directly to MySQL with ad hoc scripts.
- Do not use SQLite files or SQLite-only syntax.
- Do not directly import Raw Candidates from profile search.
- Do not use old Cycle x Route logic by default.
- Do not ask the user to choose legacy Subagent Hybrid settings.
- Do not use cross-platform evidence in v1.
- Do not import profile URLs as `video_url`.
- Do not lose `author_profile_url`; it is the KOL profile the user ultimately needs.
- Do not invent follower counts, views, emails, country, price, or engagement.
- If access to a platform is blocked, report the blocker instead of fabricating evidence.
